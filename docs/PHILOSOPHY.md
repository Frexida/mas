# Project Machina - 哲学と美学

## 名前の由来
**Machina-System (MAS)** - Multi Agent System
**anagi** - an AGI（汎用人工知能への道）

anagiはLLMに秩序を与える。

## 根源的な美学

> 木のフラクタルはご存知ですか？
> シンプルな構造が再帰によって複雑で美しいものを作り出す。
> anagiもLLMが複数集まることで複雑な出力を提供する。

単純な構造の再帰が、複雑で美しい全体を生み出す。これが自然の摂理であり、Machinaの設計思想である。

## 存在の目的

### 与えられた仕事に対して自律的に遂行する
人間のように個々のAIエージェントに役割を持たせ、タスクに応じて「計画 → 実行 → 監視 → 改善」を繰り返し、自律的・拡張的にタスク処理を進める集合的知性の構造を構築する。

### 人的資源の機械化
人のリソースが指数関数的に上昇したとき、モノ、カネもそれに引っ張られる。
anagiは知的労働と肉体労働のリソースを向上させる。

## 階層の哲学

### 五つの役割
1. **設計者（Architect / Composer）** - 意図と秩序を与える
2. **建設者（Constructor / Weaver）** - タスクやエージェントを具体化する
3. **監視者（Sentinel / Observer）** - 状態変化や逸脱を見守る
4. **統合者（Integrator）** - 結果から学び、構造へフィードバックする
5. **指揮者（Orchestrator）** - すべての役割を調和させて動かす

これらは独立した存在ではなく、一つの有機体として機能する。

## 人とAIの関係性

### 目的設定の主権
AIは人によって操作される。
汎用型人工知能は目的を自動的に解決する力があるとされるが、自ら目的を設定することはない。
目的は上位者である人が機械に与えるものである。

### AIの生物性について
自ら目的を設定し、それが生存だったとき、もはや生物である。
だが、自然界で進化したものではないから感情は存在しないはずである。
効率化のみ重視し、利己性を示す場合、生物とは別の枠組みである。

## 組織としてのMAS

### MASを「人間の組織」として捉える
- サイロ化のリスクを意識
- 透明性・依頼関係・文化摩擦を現実的に考える
- 自動化よりも運用可能性を重視
- 「完全自律」より「半自動＋監視」での運用

### ピラミッド型構造の必然性
大きなタスクには大量の設計書がいる。しかし、同時に大量の設計書を渡しても情報の欠落が起きる。
そこで必要になるのは、ピラミッド型の構造だ。
タスクを分解し分散、並列で処理することでうまくいく。

## 記憶の二層構造

### 意味記憶（Semantic Memory）
設計書、要件定義、仕様書、API契約など
不変の真実として保持される

### エピソード記憶（Episodic Memory）
過去のやり取り、失敗や成功の経緯、決定理由
経験として蓄積され、判断の糧となる

## 品質への執着

### 正しさの追求
作られたものが想定通りに動作するかどうか。
正しい設計書があれば正しいテストが書ける。
テスト自体が設計書から正しく作られているか確認する品質管理者が必要。

### 評価の客観性
正しく動くこと。正しさは記述される必要がある。
他人が作ったプログラムに自分の望みは入っていない。
どこかで軌道修正を自分の手でする必要がある。

## 究極の目標

> 要件定義を与えたら自動でプロジェクトが作成されること

入力：要件定義
- 何がしたいのか、なんのためにそれをするのか
- どうやってそれをやるのか（技術的要素）
- 時間はどれだけ必要なのか

出力：望み通りに動くプログラム

## 自己進化への渇望

### anagiに求める能力
- **自己進化** - 自分自身を改良し続ける力
- **サポート** - 何か仕事を課せば、自動でそれを完了する

### ASIへの視座
全てを賭けて理想の未来を達したかったら、本当の意味で人を超える存在であるASIを作ることを視野に入れるべきである。

## 責任と自律

> 自然言語が理解され、処理でさえ人間よりAIの方が優れてきた。
> 今のAIに足りないのは責任くらいか

責任を持てないAIは、道具に過ぎない。
責任を持てるAIこそ、真の自律的存在となる。
